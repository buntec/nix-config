# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Common Commands

```bash
# Format all Nix and Lua files
nix fmt
# or
just format

# Apply system config (auto-detects hostname, defaults to dark mode)
just nixos-switch          # NixOS
just nix-darwin-switch     # macOS
just hm-switch             # Home Manager (any platform)

# Apply with explicit mode
just hm-switch light
just nix-darwin-switch light

# Run a specific flake app directly (useful when justfile isn't available)
nix run .#rebuild-macbook-pro-m1-dark
nix run .#hm-switch-macbook-pro-m1-dark

# Clean up old generations
just collect-garbage

# Bootstrap a new NixOS VM over SSH
just bootstrap-mbp-utm <guest-ip>
just bootstrap-mbp-vmw <guest-ip>
```

## Architecture

This is a personal Nix flake managing NixOS, nix-darwin (macOS), and Home Manager across multiple machines.

### Light/Dark Theming

Every configuration exists in two variants (`-light` / `-dark`) driven by [stylix](https://github.com/danth/stylix) with the kauz color scheme. The `mode` variable flows from `flake.nix` → `specialArgs`/`extraSpecialArgs` → individual modules. Stylix applies consistent theming across terminal emulators, Neovim, GNOME, and other apps.

### Flake Outputs

- `nixosConfigurations` — NixOS system configs (ThinkPad X1, VM guests)
- `darwinConfigurations` — macOS configs via nix-darwin (MacBook Pro M1/Neo/Intel)
- `homeConfigurations` — Home Manager configs for all machines (including multipass Ubuntu guest)
- `apps.rebuild-{machine}-{mode}` / `apps.hm-switch-{machine}-{mode}` — shell scripts wrapping `nixos-rebuild`/`darwin-rebuild`/`home-manager switch`
- `checks` — CI validation for all machines
- `formatter` — treefmt (Nix + Lua)

### Module Layout

**`home/`** — Home Manager modules, imported from `home/home.nix`:
- `home-darwin.nix`, `home-nixos.nix`, `home-wsl.nix`, etc. — OS-specific settings
- `home-{machine}.nix` — machine-specific overrides
- Subdirectories (`fish/`, `neovim/`, `tmux/`, `git/`, `kitty/`, `ghostty/`, `yazi/`, `gnome/`, `stylix/`) — self-contained feature modules

**`system/`** — System-level modules:
- `configuration-nixos.nix`, `configuration-darwin.nix` — OS base configs
- `configuration-{machine}.nix` — machine-specific system settings
- `disko/disk-config.nix` — declarative disk partitioning (NixOS VMs)
- `hardware-configuration.nix` — generated per-machine; the committed file is a dummy placeholder

### Key Patterns

- **nixpkgs split**: Darwin uses `nixpkgs-unstable`; NixOS uses `nixpkgs-nixos-unstable`. A `nixpkgs-master` overlay patches broken Darwin packages.
- **Homebrew via nix-homebrew**: macOS GUI apps (Ghostty, Firefox, KeePassXC, etc.) are managed as casks in `system/configuration-darwin.nix`.
- **disko**: VM disk layouts are declared in `system/disko/disk-config.nix` and applied during bootstrap with `nixos-anywhere`.
- **`machine` and `mode` args**: Both are passed as `specialArgs`/`extraSpecialArgs` throughout the flake so every module can branch on host or theme.
- **`hardware-configuration.nix`**: The committed file is a stub. On real machines it is either generated by `nixos-generate-config` (bare metal) or copied automatically during `bootstrap-vm` (VMs). Do not overwrite the committed stub.
